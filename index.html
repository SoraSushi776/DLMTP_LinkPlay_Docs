DLMTP_LinkPlay SDK 文档

DLMTP_LinkPlay 是一个基于 WebSocket 的 Unity 联机客户端 SDK，提供完整的房间管理和 RPC 远程调用功能。

核心依赖：UnityEngine、System.Net.WebSockets | 文档版本：v2.0 | 最后更新：2026-05-17

功能特性

V WebSocket 连接管理 — 基于 ClientWebSocket 的异步连接
V 大厅 + 房间系统 — 支持创建/加入/离开房间，密码保护
V RPC 远程调用 — 4 种广播目标：全部、房主、非房主、除自己外
V 事件回调系统 — 全异步操作均有对应事件通知
V 自动心跳保活 — 可配置间隔与超时时间
V 断线自动重连 — 可配置重连次数与间隔
V 自定义房间/玩家属性 — 支持 Dictionary<string, object> 灵活扩展
V 跨平台设备信息 — 自动收集平台、设备型号、OS 信息

架构概览

┌─────────────────────────────────────────────────────┐ │ Unity 客户端 │ │ │ │ LinkPlayClient (MonoBehaviour) │ │ ├── 连接管理 (ConnectAsync / DisconnectAsync) │ │ ├── 大厅管理 (LeaveLobbyAsync) │ │ ├── 房间管理 (CreateRoom / JoinRoom / LeaveRoom) │ │ ├── 玩家信息 (UpdatePlayerInfoAsync) │ │ ├── 游戏消息 (SendGameMessageAsync) │ │ ├── RPC 系统 (InvokeRPCAsync) │ │ └── 心跳/重连 (自动) │ │ │ │ │ ▼ WebSocket │ │ ┌──────────────────────────────────────┐ │ │ │ DLMTP_LinkPlay_Socket_Server (C#) │ │ │ │ (独立控制台应用) │ │ │ └──────────────────────────────────────┘ │ └─────────────────────────────────────────────────────┘

4. RPC 属性 (LinkPlayRPC)

[LinkPlayRPC] 是一个方法级别的 Attribute，用于标记可被远程调用的方法。

定义

[AttributeUsage(AttributeTargets.Method, AllowMultiple = false, Inherited = true)]
public class LinkPlayRPC : Attribute

属性

属性名类型必填说明
Namestring否自定义 RPC 方法名。不设置时默认使用 C# 方法名。
Descriptionstring否RPC 方法描述，仅用于文档说明。

使用规则

  • 可标记在 public / private / protected 方法上
  • 方法必须位于继承自 MonoBehaviour 的类中
  • 方法所在的 GameObject 必须为 active in hierarchy
  • 参数支持：int、float、double、long、bool、string、Dictionary<string, object>
  • 不支持：GameObject、Transform 等 Unity 特有类型

6.4 公共方法

所有 async 方法均可 await，返回 bool 表示操作是否成功发送。

ConnectAsync

public async Task<bool> ConnectAsync()

连接到 WebSocket 服务器并自动加入大厅。连接后自动发送 join_lobby 请求。若已连接则直接返回 true。触发事件：成功 → OnConnected + OnJoinedLobby；失败 → OnError。

DisconnectAsync

public async Task<bool> DisconnectAsync()

主动断开连接。断开前会自动离开当前房间，然后发送 leave_lobby 请求。因为是主动断开，不触发 OnDisconnected 事件。

LeaveLobbyAsync

public async Task<bool> LeaveLobbyAsync()

离开大厅。前提：IsInLobby 必须为 true。

CreateRoomAsync

public async Task<bool> CreateRoomAsync(
    string roomName, int maxPlayers = 10,
    bool hasPassword = false, string password = "",
    Dictionary<string, object> properties = null)

创建房间。前提：已连接且在大厅中，不在房间中。maxPlayers Clamp 范围 2~50。触发事件：OnRoomCreated。

JoinRoomAsync

public async Task<bool> JoinRoomAsync(string roomName, string password = "")

加入房间。前提：已连接且在大厅中，不在房间中。触发事件：OnRoomJoined。

LeaveRoomAsync

public async Task<bool> LeaveRoomAsync()

离开当前房间。前提：必须在房间中。触发事件：OnRoomLeft。

GetRoomListAsync

public async Task<bool> GetRoomListAsync()

获取房间列表。触发事件：OnRoomListUpdated，参数为 List<RoomListItem>。

GetCurrentRoomInfoAsync

public async Task<bool> GetCurrentRoomInfoAsync()

获取当前房间详情（含玩家列表）。触发事件：OnRoomDetailInfoReceived。

UpdateRoomInfoAsync

public async Task<bool> UpdateRoomInfoAsync(Dictionary<string, object> properties)

更新房间属性（仅房主可用）。触发事件：OnRoomInfoUpdated。

UpdatePlayerInfoAsync

public async Task<bool> UpdatePlayerInfoAsync(
    string newPlayerName = null,
    Dictionary<string, object> properties = null)

更新玩家信息。如传入新名称，PlayerName 属性会同步更新。触发事件：OnPlayerInfoUpdated。

SendGameMessageAsync

public async Task<bool> SendGameMessageAsync(
    string messageType, object payload,
    List<string> targetPlayers = null)

向房间内玩家发送自定义游戏消息。targetPlayers 为 null 则广播给所有人。触发事件：目标玩家收到 OnBroadcastReceived。

InvokeRPCAsync

public async Task<bool> InvokeRPCAsync(
    string methodName, object[] parameters = null,
    RPCTargetType targetType = RPCTargetType.Others)

调用远程过程（RPC）。封装 SendGameMessageAsync，以 messageType = "rpc" 发送。触发事件：目标玩家收到 OnRPCReceived，并自动派发到 [LinkPlayRPC] 方法。

7. 消息处理器 (LinkPlayMessageHandler)

LinkPlayMessageHandler 是 internal 类，由 LinkPlayClient 内部使用，负责解析服务端返回的 JSON 消息并派发事件。

消息解析流程

原始消息 │ ├── 尝试解析为 ServerResponse（标准响应） │ ├── success=false → 触发 OnError │ └── success=true → 根据 type 分派： │ ├── join_lobby → OnConnected + OnJoinedLobby │ ├── leave_lobby → OnLeftLobby │ ├── create_room → OnRoomCreated │ ├── join_room → OnRoomJoined │ ├── leave_room → OnRoomLeft │ ├── get_room_list → OnRoomListUpdated │ ├── get_current_room_info → OnRoomDetailInfoReceived │ ├── set_current_room_info → OnRoomInfoUpdated │ └── set_player_info → OnPlayerInfoUpdated │ ├── 尝试解析为 ServerBroadcast（广播） │ └── 根据 event 值分派： │ ├── player_joined → OnBroadcastReceived │ ├── player_left → OnBroadcastReceived │ ├── room_properties_updated → OnBroadcastReceived │ ├── player_properties_updated → OnBroadcastReceived │ └── 其他 → OnBroadcastReceived │ ├── 识别为 pong → 更新心跳确认时间 │ └── 无法识别 → Debug.LogWarning

8. 协议格式

JSON 协议示例

// 客户端→服务端：加入大厅
{
    "type": "join_lobby",
    "data": "{\"secret\":\"...\",\"playerName\":\"Player1\",\"playerId\":\"...\"}",
    "timestamp": 1712345678901
}

// 服务端→客户端：响应
{
    "success": true,
    "type": "join_lobby",
    "data": "{\"playerId\":\"abc123...\"}",
    "error": null,
    "timestamp": 1712345678902
}

// 服务端→客户端：广播
{
    "type": "broadcast",
    "event": "player_joined",
    "roomId": "Room_1234",
    "data": "{\"playerId\":\"def456...\",\"playerName\":\"Player2\"}"
}

// 心跳（纯文本，非 JSON）
客户端 → 服务端: "ping"
服务端 → 客户端: "pong"

数据流时序

客户端 服务端 │ │ ├── ping ──────────────────────► │ │◄── pong ───────────────────────┤ │ │ ├── {"type":"join_lobby",...}──► │ │◄── {"success":true, │ │ "type":"join_lobby"}───────┤ │ │ ├── {"type":"create_room",...}──► │ │◄── {"success":true, │ │ "type":"create_room"}──────┤ │ │ │◄── {"type":"broadcast", │ │ "event":"player_joined"}───┤ (其他玩家加入时) │ │ ├── {"type":"send_game_message", │ │ "data":{...},...}─────────►│ │◄── (服务端转发 RPC)─────────────┤

9. 完整使用示例

9.1 基础连接与大厅

using UnityEngine;
using DLMTP_LinkPlay;

public class ConnectionManager : MonoBehaviour
{
    public LinkPlayClient client;

    private void Start()
    {
        if (client == null)
            client = gameObject.AddComponent<LinkPlayClient>();

        client.ServerSecret = "your_server_secret";
        client.PlayerName = "玩家" + Random.Range(1000, 9999);

        client.OnConnected += () => Debug.Log("已连接");
        client.OnDisconnected += () => Debug.Log("已断开");
        client.OnError += (err) => Debug.LogError(err);
        client.OnJoinedLobby += () => Debug.Log("已加入大厅");

        _ = Connect();
    }

    private async System.Threading.Tasks.Task Connect()
    {
        bool ok = await client.ConnectAsync();
        if (!ok) Debug.LogError("连接失败");
    }

    private void OnDestroy()
    {
        if (client != null) _ = client.DisconnectAsync();
    }
}

9.2 房间管理

public class RoomManager : MonoBehaviour
{
    public LinkPlayClient client;

    private void Start()
    {
        client.OnRoomCreated += (name) => Debug.Log($"房间创建: {name}");
        client.OnRoomJoined += (name) => {
            Debug.Log($"加入房间: {name}");
            _ = client.GetCurrentRoomInfoAsync();
        };
        client.OnRoomListUpdated += (rooms) => {
            foreach (var r in rooms)
                Debug.Log($"  {r.roomName} ({r.currentPlayerCount}/{r.roomMaxPlayer})");
        };
    }

    public async void CreatePublicRoom(string name)
        => await client.CreateRoomAsync(name, maxPlayers: 10);

    public async void CreatePrivateRoom(string name, string pwd)
        => await client.CreateRoomAsync(name, maxPlayers: 4,
            hasPassword: true, password: pwd);

    public async void JoinRoom(string name, string pwd = "")
        => await client.JoinRoomAsync(name, pwd);

    public async void RefreshRoomList()
        => await client.GetRoomListAsync();
}

9.3 RPC 远程调用

public class GameRPC : MonoBehaviour
{
    public LinkPlayClient client;

    [LinkPlayRPC]
    public void StartGame() => Debug.Log("游戏开始！");

    [LinkPlayRPC]
    public void PlayerScoreUpdate(string playerId, int score, int combo)
        => Debug.Log($"玩家 {playerId} 得分: {score}");

    [LinkPlayRPC(Name = "SyncHealth")]
    public void UpdateHealth(string playerId, float health)
        => Debug.Log($"玩家 {playerId} 生命值: {health}");

    // 调用 RPC
    public async void OnStartGameClicked()
        => await client.InvokeRPCAsync("StartGame",
            targetType: RPCTargetType.AllPlayersInRoom);

    public async void SyncScore(string id, int score)
        => await client.InvokeRPCAsync("PlayerScoreUpdate",
            parameters: new object[] { id, score, 0 },
            targetType: RPCTargetType.Others);

    public async void ReportToOwner(string msg)
        => await client.InvokeRPCAsync("ReceiveReport",
            parameters: new object[] { msg },
            targetType: RPCTargetType.RoomOwner);
}

9.4 玩家位置同步

public class PlayerSync : MonoBehaviour
{
    public LinkPlayClient client;
    private Vector3 _lastPos;
    private float _syncInterval = 0.1f;
    private float _lastSyncTime;

    private void Start()
    {
        client = FindFirstObjectByType<LinkPlayClient>();
        _lastPos = transform.position;
    }

    private void Update()
    {
        if (!client.IsInRoom) return;
        if (Time.time - _lastSyncTime >= _syncInterval
            && Vector3.Distance(transform.position, _lastPos) > 0.1f)
        {
            _ = SyncPosition();
            _lastPos = transform.position;
            _lastSyncTime = Time.time;
        }
    }

    private async Task SyncPosition()
    {
        var pos = transform.position;
        var rot = transform.rotation.eulerAngles;
        await client.InvokeRPCAsync("OnRemoteMove",
            parameters: new object[] { client.PlayerId,
                pos.x, pos.y, pos.z, rot.y },
            targetType: RPCTargetType.Others);
    }

    [LinkPlayRPC]
    public void OnRemoteMove(string playerId, float x, float y,
        float z, float ry)
    {
        if (playerId == client.PlayerId) return;
        var target = GameObject.Find("Player_" + playerId);
        if (target != null)
            target.transform.position = Vector3.Lerp(
                target.transform.position,
                new Vector3(x, y, z), 0.5f);
    }
}

10. 常见问题

Q: 断线后会自动重连吗？

A: 会。AutoReconnect 默认为 true。当 WebSocket 意外断开（非主动调用 DisconnectAsync()）且重连次数未达上限时，会自动尝试重连。可配置 MaxReconnectAttempts（最大重连次数，-1 为无限）和 ReconnectDelay（重连间隔秒数）。

Q: RPC 方法的参数有什么限制？

A: 参数必须是 JSON 可序列化的类型：支持 int、float、double、long、bool、string、Dictionary<string, object>。不支持 GameObject、Transform 等 Unity 特有类型，以及未标记 [Serializable] 的自定义类。

Q: RPC 方法可以是非 public 的吗？

A: 可以。DispatchRPCMethod 使用反射查找，支持 public、private、protected 方法。

Q: 如何在 RPC 中传递复杂数据？

A: 使用 Dictionary<string, object>：

var data = new Dictionary<string, object> {
    ["playerId"] = "abc123",
    ["position"] = new float[] { 1.0f, 2.0f, 3.0f },
    ["items"] = new int[] { 101, 202, 303 }
};
await client.InvokeRPCAsync("OnPlayerData", new object[] { data });

[LinkPlayRPC]
public void OnPlayerData(Dictionary<string, object> data) { /* ... */ }

Q: 心跳间隔可以修改吗？

A: 可以。在 Inspector 中修改 HeartbeatInterval（默认 30 秒）和 HeartbeatTimeout（默认 60 秒），或在代码中：client.HeartbeatInterval = 15f;

Q: 如何在 Unity 编辑器中测试联机？

A: 1) 启动服务端程序 (DLMTP_LinkPlay_Socket_Server)；2) 在 Unity 中配置 ServerUrl 为 ws://localhost:8080；3) 运行 Unity 场景，启动多个实例进行测试。

Q: OnBroadcastReceived 和具体事件有什么区别？

A: OnBroadcastReceived 是通用广播通道，接收所有服务端推送事件（玩家加入/离开、属性变更等）。OnRoomDetailInfoReceived 是特定响应通道，仅在调用 GetCurrentRoomInfoAsync() 后触发。建议：处理玩家加入/离开等广播用 OnBroadcastReceived；显示房间详情列表用 OnRoomDetailInfoReceived。

Q: ServerUrl 是如何确定的？

A: ServerUrl 是一个属性，根据 GameInfo.Instance.gameLinkPlayServerType 自动选择 Main 或 Localhost URL 地址。如需手动覆盖，直接赋值即可。
